# 页面布局优化用到的设计 Skills

这次页面优化不是单纯换颜色，而是按内容站和教程站的阅读任务来调整信息层级。

## 1. Editorial Information Hierarchy

适合教程站。页面先回答“我是否适合领取”，再进入“怎么领取”，最后补充“失败排查”和“风险提醒”。

落地位置：

- 首页 Hero 从宣传文案改成路线图式表达。
- 详情页标题下新增关键判断信息：适合人群、操作难度、优惠类型、最后验证。
- 详情页正文分成 01-05 的稳定阅读顺序。

## 2. Card as Entry Point

卡片只做摘要和入口，不承载过多正文。用户需要浏览时看卡片，需要执行时进入详情页。

落地位置：

- 首页卡片压缩信息密度。
- 卡片只保留分类、状态、耗时、摘要和关键元信息。
- 详情页承接完整操作步骤。

参考：NN/g 对 card component 的定义强调，卡片适合作为短摘要和详情入口，而不是完整内容承载。

## 3. Reading Width Control

详情页正文不铺满屏幕，避免长行阅读疲劳。

落地位置：

- 新增 `--article-max: 760px`。
- 正文段落限制在约 70ch。
- 侧栏独立为 300-340px，做领取判断和目录。

## 4. Mobile-first Article Flow

移动端先显示领取判断卡，再进入正文步骤，底部保留操作按钮。

落地位置：

- 详情页移动端侧栏上移。
- 步骤卡片单列展示。
- FAQ 默认折叠，减少滚动压力。

## 5. 8pt Spacing Rhythm

减少随机间距，统一 8、12、16、20、24、32 这一类节奏。

落地位置：

- 卡片间距、内容块间距、步骤卡片、FAQ 块统一收敛。
- 页面看起来更稳定，不再像多个组件临时拼在一起。

## 6. Risk-first UX

优惠教程不是纯导购，需要先提醒限制和风险。

落地位置：

- 详情页侧栏新增“本篇注意事项”。
- 教育邮箱类内容突出不能商用、不能买二手邮箱、不能把传闻写成可领取。
