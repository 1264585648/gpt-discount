## 2026 年 7 月 11 日核验结论

先把最容易误解的结论说清楚：

- **GitHub Student Developer Pack 仍可申请。**
- **GitHub Pro 对通过验证的学生免费。**
- **GitHub Copilot 学生新计划注册当前暂时暂停。**认证通过后，不一定能立刻开通 Copilot。
- Student Pack 是一个权益入口，不是“认证一次，所有产品自动到账”。合作伙伴优惠通常需要逐项激活。

因此，这篇教程的正确目标是：**先完成 GitHub Education 学生认证，拿到 GitHub Pro 和 Student Pack 访问资格；Copilot 则单独查看实时开放状态。**

## 一、先判断自己是否符合资格

GitHub 官方列出的核心资格是：

1. 正在就读于能够授予学历或文凭的项目，例如高中、大学、学院或符合条件的家庭教育项目。
2. 能提供证明当前学生身份的材料。
3. 拥有 GitHub 个人账号。
4. 年满 13 周岁。

非学历或非文凭项目通常不符合普通学生包资格。部分编程训练营或非正式课程，只有在机构加入 GitHub Campus Program 等情况下才可能符合条件。

## 二、材料怎么准备，才不容易被退回

材料的重点不是“看起来像学生证”，而是能明确证明你**现在仍然在读**。

### 推荐材料

- 带当前有效日期的学生证
- 当前学期课程表
- 成绩单
- 学校出具的在读或注册证明

### 图片检查清单

提交前逐项确认：

- 姓名和 GitHub 申请信息一致
- 学校名称完整可见
- 当前学期、当前学年或其他有效日期清晰可见
- 图片没有反光、遮挡、严重压缩或裁切
- 不要上传已经过期的学生证
- 不要只上传录取通知书来证明当前在读

GitHub 官方失败排查文档特别强调：材料模糊、没有当前日期，或者不能清楚显示当前学术身份，都会导致需要补充证明。

## 三、学校邮箱的正确处理方式

学校邮箱不是简单看后缀，也不是只要买到一个教育邮箱就可以。

### 已有学校邮箱

进入：

`GitHub → Settings → Emails → Add email address`

添加学校邮箱后完成邮件验证，再进入 Education Benefits 申请。

邮箱建议：

- 使用全小写格式
- 不要使用 `user+school@example.com` 这样的加号别名
- 确认邮箱没有绑定在另一个用于申请学生包的个人账号上

### 学校不提供邮箱

准备以下内容：

- 学校官网关于邮箱政策的说明；或
- 学校使用正式抬头出具的说明；以及
- 能证明当前在读的正式材料

### 学校域名不被识别

联系 GitHub Education Support 时，提供：

- 学校官方全称
- 学校官网地址
- 你正在使用的学校邮箱域名
- Education Benefits 页面出现的问题

## 四、审核通过后，怎么真正领取权益

学生认证通过后，建议按下面顺序处理。

### 1. 先确认 GitHub Pro

在 GitHub Education Portal 和账号设置中确认学生权益已生效。Student Pack 官方页面当前写明：学生在符合资格期间可免费使用 GitHub Pro。

### 2. 再进入 Student Developer Pack

打开 Pack 页面后，不要看到一个产品就立即激活。先检查：

- 是否需要额外注册合作伙伴账号
- 是否限制地区或年龄
- 是否要求信用卡
- 免费期从申请通过时开始，还是从首次激活时开始
- 到期是否自动续费或转为付费
- 云服务赠金是否排除 GPU、第三方 AI 模型或其他高成本服务

### 3. 限时权益按需激活

GitHub 官方说明，很多合作伙伴的限时优惠会从你设置或激活它时开始计算。暂时不用的权益可以先不点，避免免费期被浪费。

### 4. Copilot 单独判断

不要沿用旧教程里的“学生认证通过后直接开启 Copilot”。

截至本篇最后核验时间，Student Developer Pack 的 GitHub Copilot 条目显示：**新计划注册暂时暂停**。这属于 GitHub 当前的产品状态，不是你的学校邮箱或材料一定有问题。

## 五、失败后的排查顺序

### 情况 A：材料不清楚或没有日期

重新准备能显示当前有效日期的学生证、课程表、成绩单或在读证明，再提交。

### 情况 B：系统强制要求学术邮箱

先在 Settings → Emails 添加并验证学校邮箱。学校不发邮箱时，准备官方政策说明后再申请。

### 情况 C：学校邮箱域名不被识别

不要反复更换学校名称碰运气。记录学校官网、邮箱域名和错误页面，联系 GitHub Education Support。

### 情况 D：邮箱已被其他账号使用

停止新建账号。同一个学术邮箱不能重复申请学生包。保留已经获得权益的账号，并按官方指引合并个人账号。

### 情况 E：学生认证通过，但 Copilot 不可领取

先看 Student Pack 的 Copilot 条目。如果仍显示新计划注册暂停，就等待官方重新开放，不要重复申请学生身份。

## 六、续期与到期提醒

GitHub Education 到期后，如果仍然符合学生资格，可以回到 Education Benefits 页面重新申请。

但要注意：

- GitHub 自身学生权益与合作伙伴权益不是同一个续期规则。
- 一些合作伙伴的限时优惠只能领取一次。
- 一些优惠在首次激活后开始倒计时。
- 到期前应检查合作平台是否会转为付费或要求手动取消。
